Guide Plugin Grist

Import de donnees Excel / CSV vers Grist

Vue d'ensemble

Ce guide detaille la procedure d'installation, de configuration et d'utilisation du plugin d'import Excel/CSV pour Grist. Le plugin permet d'importer automatiquement des donnees depuis des fichiers Excel ou CSV vers vos tables Grist avec des regles personnalisees et une gestion intelligente des doublons.

Installation du Plugin

1 Ajouter une vue personnalisee
  • Cliquer sur Nouveau
  • Selectionner Ajouter une page ou une vue
  • Choisir Vue personnalisee
  • Selectionner la table qui devra etre mise a jour
  • Cliquer sur Ajouter
2 Configurer l'URL du widget
  • Entrer l'URL du plugin dans le champ prevu
  • Cliquer sur Ajouter un widget
https://france-travail.github.io/grist-widget-import-excel/
3 Autoriser l'acces
  • Cocher la case d'autorisation d'acces
  • Cliquer sur Confirmer
Important : Le plugin necessite l'acces complet au document pour fonctionner correctement (lecture et ecriture des donnees).

Configuration

Le plugin a besoin d'une table RULES_CONFIG pour stocker les regles d'import. Deux methodes sont disponibles :

Methode 1 : Creation automatique Nouveau

A Laisser le plugin creer la table automatiquement

Si la table RULES_CONFIG n'existe pas, le plugin peut la creer automatiquement :

  • Le plugin detecte l'absence de la table RULES_CONFIG
  • Il genere automatiquement la table avec toutes les colonnes necessaires
  • La premiere colonne de votre table est definie comme cle unique par defaut
  • Il ne reste plus qu'a configurer les regles via le mode administration
Recommande : Cette methode est la plus simple et evite les etapes manuelles de telechargement/import du template.

Methode 2 : Template manuel

4 Telecharger le modele de configuration
  • Verifier que toutes les colonnes a utiliser sont visibles dans la table
  • Cliquer sur Telecharger le modele RULES_CONFIG
  • Cliquer sur OK pour confirmer le telechargement
Le fichier RULES_CONFIG.xlsx sera telecharge automatiquement.
5 Importer le fichier de configuration
  • Cliquer sur Nouveau puis Importer un fichier
  • Selectionner le fichier RULES_CONFIG.xlsx
  • Importer dans une Nouvelle table
  • Cliquer sur Importer
6 Nettoyer la vue
  • Cliquer sur les 3 points de la page RULES_CONFIG creee
  • Choisir Supprimer
  • Cocher Conserver les donnees
  • Confirmer la suppression
Important : Supprimer uniquement la page et la vue, pas la table de configuration. Cochez bien "Conserver les donnees".

Acceder au mode administration

7 Ouvrir la configuration via l'engrenage Grist
  • Retourner sur la page du plugin
  • Dans le panneau lateral droit, cliquer sur l'icone engrenage du widget
  • Le mode administration s'ouvre avec les sections de configuration
Note : L'acces au mode administration se fait via l'engrenage du widget dans le panneau lateral de Grist. Seuls les editeurs du document peuvent y acceder.

Regles d'Import

Configurez le comportement d'import pour chaque colonne de votre table. Pour chaque champ, choisissez l'action a effectuer lors de l'import :

Ne jamais modifier
La valeur Grist est toujours conservee, meme si differente d'Excel.
Usage : Champs calcules ou valides manuellement
Ecraser systematiquement
La valeur Excel remplace celle de Grist a chaque import.
Usage : Donnees de reference provenant d'Excel
Mettre a jour si plus recent
La valeur Excel remplace celle de Grist uniquement si elle est plus recente.
Usage : Champs de type Date (horodatages, dates de modification)
Remplir uniquement si vide
La valeur Excel est utilisee uniquement si la cellule Grist est vide.
Usage : Valeurs par defaut, completion automatique
Ajouter si different
La valeur Excel est ajoutee a la suite si elle est differente de celle de Grist.
Usage : Champs de type Choix/Liste (commentaires, historique, tags multiples)
Colonnes formule : Les colonnes calculees (formules Grist) sont automatiquement detectees et exclues de l'import. Elles apparaissent avec la mention "(formule)" et leur selecteur est desactive.

Cle(s) unique(s) pour les doublons

Apres avoir configure les regles, definissez la ou les colonnes servant de cle unique pour identifier les doublons lors de l'import.

8 Configurer les cles uniques
  • Les modifications de regles sont enregistrees instantanement (pas de bouton de sauvegarde)
  • Cocher une ou plusieurs colonnes comme cle unique
  • Choisir le mode de cle :
Mode Composite Nouveau
Toutes les cles cochees sont combinees pour former un identifiant unique.
Exemple : Si "Nom" + "Prenom" sont cles composites, la combinaison "Dupont + Jean" identifie une ligne unique.
Mode Fallback Nouveau
Les cles sont utilisees par ordre de priorite. Si la cle 1 est vide, le plugin utilise la cle 2, etc.
Exemple : Cle 1 = "Email", Cle 2 = "Telephone". Si l'email est vide, le telephone est utilise pour identifier la ligne.
Configuration terminee ! Cliquer sur "Retour a l'import" pour quitter le mode administration et commencer a importer des donnees.

Utilisation (Mode Utilisateur)

Une fois la configuration effectuee par l'administrateur, les utilisateurs peuvent importer leurs fichiers Excel ou CSV.

1 Selectionner un fichier
  • Cliquer sur Choisir un fichier
  • Formats acceptes : .xlsx, .xls, .csv
  • Le nom du fichier s'affiche sur la bande verte une fois selectionne
2 Selectionner les onglets Nouveau

Si le fichier Excel contient plusieurs onglets, un selecteur apparait :

  • Tous les onglets sont coches par defaut
  • Decocher les onglets a ne pas importer
  • Boutons rapides : Tout selectionner / Tout deselectionner
  • L'import traitera chaque onglet selectionne de maniere sequentielle
Note : Si le fichier ne contient qu'un seul onglet, cette etape est automatiquement ignoree.
3 Verifier les correspondances
  • Cliquer sur Faire correspondre les colonnes
  • Le plugin affiche les correspondances Excel ↔ Grist detectees automatiquement
  • Les colonnes sans correspondance sont signalees par "Aucun match"
  • Un rapport de validation s'affiche avec les eventuels avertissements
4 Simuler l'import (optionnel) Nouveau
  • Cliquer sur Simuler l'import pour tester sans modifier les donnees
  • Le rapport affiche les actions qui seraient effectuees (ajouts, mises a jour, ignorees)
  • Permet de verifier le resultat avant de lancer l'import reel
5 Lancer l'import
  • Cliquer sur Lancer l'import
  • Une barre de progression s'affiche en temps reel
  • A la fin, un recapitulatif detaille les lignes ajoutees, mises a jour, ignorees et en erreur
6 Annuler le dernier import (optionnel) Nouveau
  • Apres un import reussi, le bouton Annuler le dernier import apparait
  • Cliquer dessus et confirmer pour restaurer les donnees a leur etat precedent
  • Les lignes ajoutees sont supprimees et les lignes modifiees sont restaurees
  • Le rollback couvre tous les onglets d'un meme import
Attention : Le rollback n'est disponible que pour le dernier import de la session en cours. Si vous fermez ou rechargez le widget, le rollback n'est plus possible.

Table IMPORT_LOG

Chaque import (et simulation) est automatiquement enregistre dans une table IMPORT_LOG creee automatiquement par le plugin. Elle contient :

  • Horodatage de l'import
  • Nom du fichier et de l'onglet importe
  • Statistiques : lignes ajoutees, mises a jour, ignorees, erreurs
  • Indicateur de simulation (dry-run) ou import reel
  • Indicateur d'annulation (rolled back)
